# SemanticServer (Console Edition)

Ультракомпактный семантический сервер на Free Pascal (Lazarus). Оптимизирован для работы на Edge-устройствах и Jetson Nano.

---

## 🚀 Ключевые особенности

* **Минимализм:** Исполняемый файл весит всего 3.5 Мб. Включает в себя HTTP-сервер и драйверы БД.

* **Алгоритм «Эстафета хвоста»:** Ядро обхода графа реализовано в 30 строках кода без использования рекурсии.

* **Оптимизация памяти:**
    * Использование динамических массивов целых чисел (`array of Integer`) для стека вместо медленных строк.
    * Использование `TStringBuilder` для формирования HTML (исключает фрагментацию памяти).

* **Многопоточность:** Обработка запросов в режиме `Threaded := True`. Работа с SQLite в режиме WAL.

---

## 🏗 Архитектура проекта

### 1. Главный модуль (semanticserver.lpr)
Является точкой входа и сетевым диспетчером.
* Инициализирует объект базы данных при старте.
* Настраивает маршрутизацию: `/` (Index) и `/forum` (Data).
* Управляет жизненным циклом потоков-воркеров.

### 2. Модуль базы данных (unit_db.pas)
Адаптер для работы с SQLite.
* Реализует атомарное чтение данных (ID, Контент, Хронология).
* Содержит настройки стабильности для Linux (PRAGMA busy_timeout).

### 3. Модуль логики (unit_worker.pas)
Сердце системы, где реализован алгоритм обхода.
* **Навигатор:** Основной цикл, использующий массив как LIFO-стек.
* **ArtistDispatcher:** Метод-распределитель, отделяющий графику (Artist) от текста (Viewer).
* **RenderNodeHTML:** Изолированная процедура формирования дизайна карточек.

## ⚡ Аппаратная оптимизация: Разделение I/O потоков

Для достижения максимальной отзывчивости на системах с ограниченной пропускной способностью шины (Jetson Nano), в проекте будет реализована концепция **физического разделения хранилищ**:

* **Диск А (Системный/Текстовый):** Выделенный канал для базы данных SQLite. Обрабатывает сотни мелких запросов чтения/записи (авторизация, сессии, извлечение хронологии и текста). Оптимизирован под минимальные задержки (Latency).
* **Диск Б (Медиа/Графический):** Выделенный физический интерфейс (отдельный USB-порт) для хранения тяжелых графических ассетов и сэмплов. 
* **Синхронный рендеринг:** Такая конфигурация позволяет "Художнику" масштабировать изображения и формировать глобальную карту визуализации параллельно с "Эстафетой" текста, исключая конкуренцию за головку диска или контроллер памяти.

Это решение предотвращает эффект "I/O Wait" и гарантирует плавную работу интерфейса даже при высокой нагрузке на семантическое ядро.


---

"Для максимальной утилизации аппаратных ресурсов Jetson Nano, тяжелый графический поток (масштабирование сэмплов и сборка глобальной карты) будет перенесен на выделенную шину PCIe Gen2 x1 через слот M.2 Key E, что обеспечивает прямой доступ к памяти (DMA). Текстовое семантическое ядро (БД SQLite) изолировано на шине USB 3.0. Это гарантирует нулевую конкуренцию за ввод-вывод между логикой сервера и графическим движком."

### Как запустить

1. Скомпилируйте проект: `fpc semanticserver.lpr`
2. Положите файл `forum.db` в папку с программой.
3. Запустите: `./semanticserver`
4. Браузер: `http://localhost:8080/forum`

## 🧩 Философия данных: Таблица как Дерево

Философия данных базируется на концепции, что любая двумерная таблица является частным случаем древовидной структуры. 

* **Геометрия:** Ответы на корень формируют ось X (ширина/братья), а вложенные ответы — ось Y (глубина/потомки).
* **Маркер Хронологии:** Для обеспечения молниеносной навигации в строку `Chrono` дописывается порядковый номер узла среди "старших братьев". 
* **Карта ID:** Система оперирует "Картой таблицы", где логическая структура хранится как дерево с идеальными ID, а в его ячейках содержатся фактические физические ID узлов. 

Это позволяет "схлопывать" многомерные семантические связи в эффективный линейный поток данных, сохраняя при этом возможность мгновенного извлечения.
